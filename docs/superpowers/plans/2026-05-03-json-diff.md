# JSON Diff Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the JSON Diff placeholder with a fully functional semantic diff tool: two CodeMirror editors (left + right), a Rust diff engine that ignores object key order and compares arrays positionally, a virtualized result tree with status-driven coloring (Equal / Added / Removed / Modified / TypeChanged), default-collapse identical subtrees, "show only differences" toggle, search by path, and history save/load with `{ left, right }` payload.

**Architecture:** All diff computation happens in Rust (`json_diff` command building a `DiffTree`). Both inputs are first parsed via `serde_json::Value`, then walked recursively to produce a unified diff tree where each node carries a `DiffStatus`. The frontend stores `DiffTree` in a Zustand slice and renders a flattened-list virtualized tree using the same `react-window` + closer-row pattern as the JSON Viewer. The two editors reuse the existing CodeMirror wrapper from the JSON Viewer (extracted to a shared location).

**Tech Stack:** serde_json (existing) · `DiffTree` / `DiffStatus` domain types (new) · CodeMirror 6 (existing) · `react-window` `VariableSizeList` (existing) · Zustand (existing) · existing i18n + AppError + history infrastructure.

**Granularity:** This plan groups work into **5 coarse tasks**, each completable in a single subagent dispatch + review pass. No micro-task per file. Each task does its own TDD where it writes new domain logic.

**Type surface (Rust):**

```rust
// src-tauri/src/domain/diff.rs
pub struct DiffTree { pub root: DiffNode, pub stats: DiffStats }
pub struct DiffStats { pub total_nodes: u32, pub differences: u32 }

#[serde(tag = "kind", rename_all = "snake_case")]
pub enum DiffKey { Root, Object { name: String }, Array { index: u32 } }

#[serde(tag = "type", rename_all = "snake_case")]
pub enum DiffValue {
    Null,
    Bool { value: bool },
    Number { raw: String },
    String { value: String },
    Object { key_count: u32 },
    Array { item_count: u32 },
}

#[serde(tag = "status", rename_all = "snake_case")]
pub enum DiffStatus {
    Equal { value: DiffValue },
    Added { right: DiffValue },
    Removed { left: DiffValue },
    Modified { left: DiffValue, right: DiffValue },
    TypeChanged { left: DiffValue, right: DiffValue },
}

pub struct DiffNode {
    pub id: u32,
    pub key: DiffKey,
    pub path: String,
    #[serde(flatten)] pub status: DiffStatus,
    pub children: Vec<DiffNode>,
    pub has_difference: bool,
}

pub fn build_diff(left: Value, right: Value) -> DiffTree;
pub fn build_diff_from_strings(left: &str, right: &str) -> Result<DiffTree, AppError>;
```

**Algorithm notes:**
- Objects: take `union(left.keys, right.keys)`, sort, recurse on each. Container itself is `Equal` even when children differ (the difference rolls up via `has_difference`).
- Arrays: positional zip up to `max(left.len, right.len)`. Missing-on-right → `Added`. Missing-on-left → `Removed`. Both present → recurse.
- When a whole subtree is `Added` (or `Removed`), build child `Added`/`Removed` nodes recursively for that side so the UI can render the subtree if the user expands it.
- Path uses the existing `domain::path::{join_object_key, join_array_index}` helpers.
- Container × non-container at the same key → `TypeChanged` at the container node, no recursion below.

**Procedure:**
1. Create branch: `git checkout main && git checkout -b plan-3-json-diff`.
2. Sanity-check baseline: `cargo test` + `npx vitest run` both green.
3. Add `pub mod diff;` to `src-tauri/src/domain/mod.rs`.
4. Create `src-tauri/src/domain/diff.rs` with the type definitions above and a stub `build_diff` returning `unimplemented!()`.
5. Drive the implementation via TDD: write tests for each case, then implement until they pass. Suggested order — primitives, then object union, then array positional, then nested propagation, then stats, then `build_diff_from_strings`, then serde-flatten roundtrip.

   use crate::error::AppError;

   pub async fn json_diff(left: String, right: String) -> Result<DiffTree, AppError> {
       tauri::async_runtime::spawn_blocking(move || build_diff_from_strings(&left, &right))
           .await
           .map_err(|e| AppError::Internal(format!("spawn_blocking join error: {e}")))?

**Type surface (TS — append to `src/types/ipc.ts`):**

```ts
export type DiffKey =
  | { kind: "root" }
  | { kind: "object"; name: string }
  | { kind: "array"; index: number };

export type DiffValue =
  | { type: "null" }
  | { type: "bool"; value: boolean }
  | { type: "number"; raw: string }
  | { type: "string"; value: string }
  | { type: "object"; key_count: number }
  | { type: "array"; item_count: number };

export type DiffStatus =
  | { status: "equal"; value: DiffValue }
  | { status: "added"; right: DiffValue }
  | { status: "removed"; left: DiffValue }
  | { status: "modified"; left: DiffValue; right: DiffValue }
  | { status: "type_changed"; left: DiffValue; right: DiffValue };

// Status fields are flattened onto DiffNode (matches #[serde(flatten)] in Rust).
export type DiffNode = {
  id: number;
  key: DiffKey;
  path: string;
  children: DiffNode[];
  has_difference: boolean;
} & DiffStatus;

export interface DiffStats { total_nodes: number; differences: number }
export interface DiffTree { root: DiffNode; stats: DiffStats }
```

**Store shape (TS):**

```ts
interface JsonDiffState {
  left: string; right: string;
  tree: DiffTree | null;
  diffError: { line: number; col: number; message: string } | null;
  showOnlyDiffs: boolean;          // default true
  collapseSet: Set<number>;
  searchQuery: string;
  loadedHistoryId: number | null;
  savedLeft: string | null;        // for dirty detection (mirror JSON Viewer pattern)
  savedRight: string | null;
  setLeft / setRight / setShowOnlyDiffs / setSearchQuery / toggleCollapse;
  swap(): void;
  clear(): void;
  compare(): Promise<void>;        // calls diffApi.diff; handles IpcError parse code
  setLoadedHistoryId / setSaved(left,right);
  isDirty(): boolean;              // any side diverges from its saved baseline
}
```

**Flatten shape (TS):**

```ts
interface DiffFlattenOptions {
  collapseSet: Set<number>;
  showOnlyDiffs: boolean;
  searchQuery: string;             // path-substring filter; empty = no filter
}
interface VisibleDiffNode {
  node: DiffNode;
  depth: number;
  isCollapsed: boolean;
  isExpandable: boolean;
  closer?: "}" | "]";
}
function flattenDiff(root: DiffNode, opts: DiffFlattenOptions): VisibleDiffNode[];
```

**Closer rows:** emit a synthetic row after children of expanded objects/arrays — same convention as the JSON Viewer's `flatten`. A closer's `node` references the OPENER (so search ancestor logic can pair them by id). Empty containers don't get closers.
